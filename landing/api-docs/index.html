Quick Start

Top up your wallet, launch a campaign targeting new users, then track conversions. Three steps, three languages.

# 1. Top up your merchant wallet (USD)
curl -X POST https://api.letskix.com/api/v1/wallet/brand_acme/topup \
  -H "Authorization: Bearer $KIX_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"amount_usd": 500, "payment_method": "stripe_card"}'

# 2. Create a new-user acquisition campaign
curl -X POST https://api.letskix.com/api/v1/campaigns/create \
  -H "Authorization: Bearer $KIX_API_KEY" \
  -d '{
    "brand_id": "brand_acme",
    "name": "Spring NewUser Push",
    "target_audience": "new_users_only",
    "bid_usd": 0.42,
    "daily_budget_usd": 50
  }'

# 3. Track conversion when user redeems
curl -X POST https://api.letskix.com/api/v1/attribution/track/conversion \
  -H "Authorization: Bearer $KIX_API_KEY" \
  -d '{"kid": "kid_abc123", "event": "purchase", "value_usd": 12.50}'

Authentication

Three auth flavours depending on which audience the endpoint serves.

Merchant Bearer Token Server

Long-lived API keys minted in the Brand Portal. Required for campaigns, wallet, attribution, audiences, and most write endpoints. Send as Authorization: Bearer kix_live_xxx.

KiX ID Session (kid) User

Short-lived JWT representing an end-user (kid_xxx) — used by the KiX App, Pixel SDK, and merchant-embedded SDKs. Issued by /api/v1/kix-id/session and refreshed automatically.

Pixel ID Client

Public, low-trust ID embedded in client JS for event collection only. Cannot read data; can only write conversions/page-views via /api/v1/pixel/collect. Rate-limited per origin.

Common Error Codes

All errors return JSON with {detail, code, request_id}. Always log request_id when filing a support ticket.

HTTPcodeWhen it fires
400invalid_requestMalformed JSON, missing required field, or schema mismatch.
401auth_requiredMissing or expired Bearer token / kid session.
403scope_forbiddenToken lacks the role/scope (RBAC) needed for this resource.
404not_foundBrand, campaign, kid, or voucher id does not exist.
409conflictDuplicate idempotency key or state-machine collision (e.g. double redemption).
422validation_failedField-level validation error — see detail[].loc for the failing path.
429rate_limitedPer-token or per-origin quota exceeded. Honor Retry-After header.
451consent_requiredEnd-user has not granted the required GDPR / PIPL consent scope.
500internal_errorUnexpected server error — report request_id to support.
503service_unavailableDownstream (Redis/auction engine) temporarily degraded. Safe to retry.
