How a Trinity 3T sim round works

① Render

Playwright loads the actual landing page in headless Chromium with the persona's locale + timezone. Post-JS DOM is captured (not raw HTML — so JS-driven i18n, currency, and dynamic widgets are applied).

② Simulate

The rendered DOM + 5KB persona profile + 8KB context is sent to Claude Sonnet 4.5 via OpenRouter. The model role-plays the user and returns a 7-section walkthrough (first reaction → final verdict).

③ Compare

The verdict is logged with timestamp and commit hash. The next iteration's verdict is compared to the previous. Verbatim quotes are pulled into the Bible and into this page — including the ones that don't flatter us.
