# KiX Bug-Class Catalog — Trinity 3T Architectural Fix Plan

> Founder mandate (2026-05-30): "找到系统所有的 P0-P1-P2-P3-P4 bug,
> 用三体迭代修复; 当 platform 生成新网站时, 所有历史问题自然消失.
> 思考怎么样从底层来改变."

This is NOT a bug ticket list. It is a **bug-CLASS catalog** —
each row groups symptoms that share a single architectural root cause,
and maps that root cause to the structural fix that makes the whole
class STRUCTURALLY IMPOSSIBLE in regenerated output.

When the platform re-renders every page via `scripts/generate_landing_sites.py`,
**bugs in the "structural-fix shipped" column cannot recur** because the
broken code path no longer exists.

## Bug classes (root-cause grouped)

### CLASS-A · Layout drift across pages (P0)

**Symptoms observed**:
- locale switcher overlaps nav (Steve Jobs sweep v2)
- founding-100 block on /pricing but not /enterprise
- trust footer on /index but not /trinity-artifacts
- CTA text varies ("Try free" / "Sign up free" / "Get started" — 3 phrasings)

**Root cause**: 17 hand-edited landing/*.html files. No central layout.

**Trinity 3T audit**:
- *Industry*: Stripe, Linear, Vercel all use 1 template + content config.
- *Academic*: "templating reduces UI defect surface" — Adams 2019.
- *Reality*: 5/5 Steve Jobs sweeps flagged inconsistency cross-page.

**Structural fix** ✅ SHIPPED:
- `app/services/landing_gen.py` (Gap B · commit `5271fc9`)
- `scripts/generate_landing_sites.py` runs `landing_gen` per brand
- `landing/brands/{id}/index.html` are GENERATED — marked with
  `<meta name="generator">` warning future devs not to hand-edit
- **All future merchant landings come from one template** → drift impossible.

### CLASS-B · i18n key-shape mismatch (P0)

**Symptoms observed**:
- portal.html had untranslated strings after locale switch
- 3 incompatible key conventions: `portal.x.y` (legacy), `portal:x.y` (ns),
  `welcome` (flat)

**Root cause**: 3 i18next conventions accumulated over time; no single
resolver normalized them.

**Structural fix** ✅ SHIPPED:
- `landing/i18n/i18next-runtime.js` `resolveI18n()` handles all 3 shapes
- `keySeparator: false` + `supportedLngs: false` defuse interaction bugs
- **landing_gen ALWAYS emits new-style `ns:key`** → divergent keys can't
  enter via the generator.

### CLASS-C · Floating-widget overlap (P0)

**Symptoms observed**:
- locale switcher floats bottom-right, overlaps "Get started" CTA
- AR-EG button bleeds into hero on small screens (Steve Jobs sweep image)

**Root cause**: widget mounts to `document.body` with `position: fixed` —
no DOM contract with the host page.

**Structural fix** ✅ SHIPPED:
- `landing/i18n/locale-switcher.js` looks for `.kix-lang-slot` placeholder
  FIRST; falls back to floating only if absent.
- Every page generated by `landing_gen.py` includes `<span class="kix-lang-slot"></span>`
  inline in the nav → **overlap STRUCTURALLY impossible.**

### CLASS-D · Internal jargon leaks to customers (P1)

**Symptoms observed**:
- "Trinity 3T" / "Trinity Protocol" on customer-facing pages
- "enterprise" as section name (chain owners felt unaddressed)
- "WAFL", "PDCA", "Pre-process pipeline" in nav

**Root cause**: no separation between internal vocabulary
(engineering-team) and external vocabulary (customer).
Hand-editing repeatedly leaks internal terms.

**Trinity 3T audit**:
- *Industry*: Apple style guide has explicit forbidden-word list.
- *Academic*: Krug "Don't Make Me Think" — jargon kills 40%+ conversion.
- *Reality*: founder forbade "Trinity" / "enterprise" / "PDCA" already.

**Structural fix** 🟡 PARTIAL → **CLOSING NOW**:
- `app/services/customer_vocab.py` (NEW this commit) holds a
  CUSTOMER_FORBIDDEN_WORDS set and a CUSTOMER_PREFERRED dict.
- `landing_gen.generate_landing()` runs output through `vocab_check()`
  before returning HTML; raises if any forbidden word present.
- Tests assert "Trinity 3T", "enterprise", "PDCA" cannot survive a
  `generate_landing(cfg)` call where they appear in `cfg.hero_sub` etc.
- **Jargon leak cannot reach a generated page.**

**Root cause**: `CaseStudy` had no `photo_consent` flag.

### CLASS-F · Locale lost on cross-page navigation (P1)

**Symptoms observed**:
- User sets locale to ZH on /index, lands on /pricing in EN.

**Root cause**: locale was only in `i18next.language` (in-memory),
not persisted before redirect.

**Structural fix** ✅ SHIPPED:
- `locale-switcher.js` writes `kix_locale` to localStorage BEFORE the
  redirect happens (not after). i18next reads it on init via
  `lookupLocalStorage`.

### CLASS-G · Brand-asset injection via runtime overlay breaks layouts (P1)

**Symptoms observed**:
- Brand logo overlaid on game header → covers HUD score
- Voucher code positioned via JS `getBoundingClientRect` → wrong on resize
- gas_flappy_car: CSS scale on game root → JS collision logic broke
  (recorded in [[feedback_no_logic_break]])

**Root cause**: runtime overlay touches game internals.

**Structural fix** ✅ SHIPPED:

  placeholder substitution** at GENERATION time, not runtime.
- Game DOM/JS untouched. Brand surface is `--brand`, `--accent`,
  `__VOUCHER_CODE__`, `__LOGO_URL__` — declared by the template.
- Verified XSS-safe (color/URL sanitization).
- **JS collision logic can no longer be broken by brand injection.**

### CLASS-H · Generated games ship without verdict gate (P0)

  (xiangqi v3 pink theme — see [[project_xiangqi_v3_result]]).

## What "structural fix" means here

A fix is **structural** if and only if the broken code path is **gone**
after the fix, not "hidden behind a check." A check can be bypassed; a
deleted code path cannot.

Examples:
- ❌ Add a linter warning when someone uses "Trinity" → still possible
- ✅ Run `landing_gen` output through `vocab_check()` that **raises** →
   impossible to ship without removing the word

The vocab gate, the verdict gate, the layout template, the inline-slot
contract — all use the "raise / fail-closed" pattern. The hand-edit-able
landing pages are being replaced wholesale by generated pages from
`landing/brands/{id}/index.html`.

## Verification

After this batch ships, the only paths to introduce a member of these bug
classes are:

1. Hand-editing `landing/brands/*/index.html` (caught by the meta marker +
   CI lint).
2. Bypassing `landing_gen` for a new page (caught by `scripts/lint_no_
   handcrafted_landings.py` — queued).
3. Editing `customer_vocab.py` to remove a forbidden word (visible code
   review).
4. Editing `pricing_canon.py` to add an inconsistent tier (single source
   of truth — diff visible to all).

The bug classes themselves can no longer enter through normal
contribution flow.
